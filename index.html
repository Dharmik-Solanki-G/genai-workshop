LLMs: How Do They Actually Work? üß†

üéì Karpathy's "Build GPT from Scratch" Key Points

  • Tokenization: Text ‚Üí numbers (tokens) that models understand
  • Embeddings: Tokens ‚Üí vectors capturing meaning
  • Attention: "What words should I pay attention to?"
  • Training: Predict next token millions of times

References: Build GPT | How GPT Works

# Mini-Transformer Forward Pass 1. Take input text 2. Convert to tokens 3. Create embeddings 4. Apply attention (Q, K, V) 5. Feed through neural network 6. Predict next token 7. Sample from probabilities
# Simplified transformer step import torch import torch.nn as nn class MiniTransformer(nn.Module): def __init__(self, vocab_size, d_model): super().__init__() self.embed = nn.Embedding(vocab_size, d_model) self.attention = nn.MultiheadAttention(d_model, num_heads=8) self.ffn = nn.Linear(d_model, vocab_size) def forward(self, x): x = self.embed(x) # tokens to embeddings x, _ = self.attention(x, x, x) # self-attention return self.ffn(x) # predict next token

üéØ Why RAG Matters

  • Freshness: Access to latest information beyond training cutoff
  • Grounding: Reduce hallucinations with real sources
  • Efficiency: Smaller models can access vast knowledge
  • Privacy: Keep sensitive data out of training

References: RAG Explained | My RAG Demo

# RAG Pipeline 1. User asks: "What's our Q3 revenue?" 2. Search vector DB for relevant docs 3. Re-rank by relevance score 4. Pack top results into context 5. Generate answer with citations 6. Return: "Q3 revenue was $2.3M [source: report.pdf]"
from langchain.vectorstores import FAISS from langchain.embeddings import OpenAIEmbeddings # Simple RAG def rag_query(question, documents): # Create vector store vectorstore = FAISS.from_documents( documents, OpenAIEmbeddings() ) # Retrieve relevant chunks docs = vectorstore.similarity_search(question, k=3) # Augment prompt with context context = "\n".join([d.page_content for d in docs]) prompt = f"Context: {context}\n\nQuestion: {question}" return llm.generate(prompt)

Mini-Lab: Build Your First Agent üèóÔ∏è

Goal: Document Q&A Agent

Build an agent that answers questions about any PDF/text file

# Install requirements pip install langchain openai chromadb pypdf # Import libraries from langchain.document_loaders import PyPDFLoader from langchain.text_splitter import RecursiveCharacterTextSplitter from langchain.embeddings import OpenAIEmbeddings from langchain.vectorstores import Chroma
# Load and process document loader = PyPDFLoader("document.pdf") documents = loader.load() # Split into chunks text_splitter = RecursiveCharacterTextSplitter( chunk_size=1000, chunk_overlap=200 ) chunks = text_splitter.split_documents(documents) # Create vector store embeddings = OpenAIEmbeddings() vectorstore = Chroma.from_documents(chunks, embeddings)
# Create QA agent from langchain.chains import RetrievalQA from langchain.llms import OpenAI qa_agent = RetrievalQA.from_chain_type( llm=OpenAI(temperature=0), chain_type="stuff", retriever=vectorstore.as_retriever() ) # Ask questions result = qa_agent.run("What is the main topic of this document?") print(result)
